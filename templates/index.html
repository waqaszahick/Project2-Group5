Our Objective

Our main objective in making this application is to provide people with an easy and accessible way to find different foods based on their location, budget and personal preferences. This one-stop-shop application means that people can browse for restaurant options in a convenient way wherever they may be.

In this project, we have allowed the user(s) to find the restaurant(s) of their choice via the following main three ways:


  • By name - finding restaurant(s) by name
  • By state / territory - finding restaurant(s) by state or territory
  • By city - finding restaurant(s) by city

Some of the additional categories through which user(s) would be able to find restaurant(s) are:


  • Cuisine Type - such as arabian, italian, halal, vegetarian etc.
  • Rating - the individual star-rating of restaurants
  • Delivery Services - food delivery services such as 'Uber Eats'

Solution

The solution was to scrape data found on the internet for restaurants in Australia. The data is then cleansed and stored in such a way that makes it as accessible as possible. The way this data is stored will allow for analysis on correlations.

Extraction

Our primary datasources were 'Uber Eats' website and 'GooglePlace' API, which provided national and state-level data in CSV and JSON formats.

Transformation

The 'GooglePlace' API response is converted from JSON datatype and extracted to the CSV file while 'Uber Eats' data was taken using 'beautifulsoup' techniques.

Loading

We used 'MongoDB' to save and load the real-time data using 'Flask' API and Python 'MongoDB' libraries.

Output

The output of this project is various plots, figures and other visualisation tools to easily view and understand all of the data on your favourite restaurants.

Plotting

By using various Javascript libraries including Leaflet, D3, and Chart.js we plotted data from our MongoDB, with the objective of getting insight into the data

Analysis

After visualizing the data we looked to determine what the insights informed us about our project.

Visualization

We created this site to outline our findings through HTML/CSS pages and JavaScript to query our MongoDB database through a flask application.

Key Observations:

Following are some of the key trends that we observed while working on this project:

The State of Queensland has the highest number of restaurants in Australia according to this dataset.

The cities of Gold Coast (in QLD) and Wollongong (in NSW) have the highest number of restaurants in Australia according to this dataset.

'Pins Kitchen' (7/47 Ashmore Road, Bundall, Gold Coast, Queensland) is the highest rated restaurant in Australia with 5.0 stars.

'Uber Eats' is used mostly used for food delivery services in Australia.
